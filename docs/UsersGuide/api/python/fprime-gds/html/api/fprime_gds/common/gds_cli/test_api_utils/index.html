fprime_gds.common.gds_cli.test_api_utils.initialize_test_api(app_dictionary_path: str, log_path: str = None, server_ip: str = '127.0.0.1', server_port: int = 50050) → IntegrationTestAPI[source]¶

Initializes an Integration Test API instance for use.

Note that these MUST be manually disconnected when you’re done using them, by calling “pipeline.disconnect()” and “api.teardown()”, respectively.

Parameters

  • app_dictionary_path – A string path from the current working directory to the “<project name>AppDictionary.xml” file for the project you’re using the API with

  • log_path – A string path to where files should be logged, or “None” if you only want console output

  • server_ip – The IP of the GDS server you want to connect to

  • server_port – The port for the Test API on the GDS server

fprime_gds.common.gds_cli.test_api_utils.get_upcoming_event(test_api: IntegrationTestAPI, search_filter: predicates.predicate, start_time='NOW', timeout: float = 5) → EventData[source]¶

Returns the next event matching the given search filter that occurs after this is called. Times out after the given amount if no matching new events are found.

Parameters

  • test_api – An API instance that will be called to find the next event

  • search_filter – A predicate each found event is tested against; if the event doesn’t test “True” against this, we ignore it and keep searching

  • start_time – An optional index or predicate to specify the earliest event time to search for

  • timeout – The maximum time (in seconds) to wait for an event

Returns

The first “EventData” found that passes the filter, or “None” if no such event is found within time

fprime_gds.common.gds_cli.test_api_utils.get_item_list(item_dictionary: Dict[Any, DataTemplate], search_filter: predicates.predicate, template_to_data: types.FunctionType) → List[SysData][source]¶

Returns an ID-sorted list containing all the possible item types that could occur on the running F’ instance, and information about each one.

Parameters

  • item_dictionary – A dictionary full of DataTemplate objects containing the events you want to list

  • search_filter – A Test API predicate used to filter out which items to include in the returned list

  • template_to_data – A callback function accepting a single DataTemplate object and returning a SysData object

Returns

An ID sorted list with all the items from the given dictionary AFTER being filtered
